## 1. 项目概述

### 1.1 项目背景

解决方案团队需要一套智能工作记录和统计系统，包含约10名解决方案经理及2名解决方案组长。

### 1.2 核心目标

1. 解决方案经理以自然语言进行工作记录，系统智能分析并自动填入工作统计表
2. 解决方案组长可查看团队工作记录及统计表
3. 解决方案组长可统计、管理团队工作量

### 1.3 用户角色

| 角色 | 人数 | 职责 |
|------|------|------|
| 解决方案经理 | ~10人 | 提交工作记录、导出个人记录 |
| 解决方案组长 | 2人 | 查看团队记录、统计数据、配置权重、配置AI大模型 |

## 4. 核心模块

### 4.1 账号管理

- 组长邀请码机制：任何人可通过邀请码注册成为解决方案组长
- 第一个组长使用预设邀请码（环境变量配置）注册
- 后续组长由现有组长在后台生成邀请码并分享
- 组长创建账号：组长在管理后台创建团队成员账号，并指定角色（解决方案经理 / 解决方案组长）
- 成员登录后仅可查看和修改自己的数据

**邀请码机制：**
- 预设邀请码（部署时配置）：用于第一个组长注册
- 组长可在后台生成新的邀请码
- 邀请码有时效性（24小时）
- 被邀请人注册时输入邀请码，自动成为组长

**账号创建流程：**
1. 任何人通过邀请码注册账号
2. 组长登录管理后台（首个组长注册后）
3. 新增成员账号（邮箱、姓名、角色、初始密码）
4. 成员使用初始密码登录，可自主修改密码

### 4.2 客户档案模块

**功能：**
- 客户名称必须为全称（官网/企查查等为准）
- 存入数据库，工作记录填报时只能点选
- 智能匹配：支持简称、相似名称匹配
- 新客户必须先建档才能填报工作

**表结构：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| name | text | 客户全称 |
| industry | text | 行业分类 |
| created_by | uuid | 创建人 |
| created_at | timestamp | 创建时间 |

### 4.3 工作记录模块

**输入模式：**
- 自然语言输入为主：用户输入描述文字，AI自动解析
- 引导补充为辅：信息不足时提示用户补充

**工作统计表字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| user_id | uuid | 提交人 |
| project_name | text | 项目/商机 |
| customer_id | uuid | 客户（关联客户档案） |
| industry | text | 行业分类（文旅/住建/传媒/体育） |
| stage | text | 进展阶段 |
| customer_manager | text | 客户经理 |
| support_role | text | 支撑角色（一线支撑/二线支撑） |
| support_units | text[] | 调用支撑单位（多选） |
| work_content | text | 工作事项 |
| work_date | date | 工作日期 |
| work_categories | text[] | 工作分类（多选） |
| work_weight | integer | 工作权重（自动计算） |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

**行业分类（单选）：** 文旅 / 住建 / 传媒 / 体育

**进展阶段（单选）：**
- 方案阶段
- 招投标过程
- 已签合同
- 项目暂停
- 项目关闭

**支撑角色（单选）：**
- 一线支撑：作为主要解决方案经理，直接负责本项目
- 二线支撑：协助项目牵线、对接资源，或提供公司成品材料

**调用支撑单位（多选或不涉及/无）：**
- 数智北分 / 云北分 / 云中台 / AI团队 / 专业公司
- 不涉及（无需协作）
- 无（涉及但未调用）

**工作分类（多选）：**
- 内部部门需求对接
- 生态交流
- 简单方案
- 复杂方案
- 日常方案汇报
- 客户简单交流
- 招投标
- 流程支撑
- 方案审核
- 培训
- 内部会议
- 高层汇报/展厅讲解

### 4.4 工作权重规则

**权重映射：**

| 工作分类 | 权重 | 全局可调整 |
|---------|------|-----------|
| 内部部门需求对接 | 1 | ✅ |
| 生态交流 | 1 | ✅ |
| 简单方案 | 2 | ✅ |
| 复杂方案 | 5 | ✅ |
| 日常方案汇报 | 3 | ✅ |
| 客户简单交流 | 2 | ✅ |
| **招投标** | **5** | **❌（按人单独配置）** |
| 流程支撑 | 1 | ✅ |
| 方案审核 | 1 | ✅ |
| 培训 | 1 | ✅ |
| 内部会议 | 1 | ✅ |
| 高层汇报/展厅讲解 | 5 | ✅ |

**规则：**
- 工作权重与工作分类绑定，分类确定后自动带出，不可手动修改
- 组长可全局调整权重（除招投标外）
- 招投标权重由组长针对每个解决方案经理单独配置

### 4.5 统计模块

**时间维度：**
- 按周
- 按月
- 按年
- 自定义时间周期

**人员维度：**
- 全选（统计全部人员）
- 多选（选择指定人员）

**统计内容：**
- 工作记录数量
- 工作权重汇总
- 各分类工作量分布

### 4.6 导出模块

| 角色 | 可导出内容 |
|------|-----------|
| 解决方案经理 | 自己的工作记录 |
| 解决方案组长 | 自己的工作记录 + 统计信息 |

**导出格式：** Excel (.xlsx)

### 4.7 大模型配置模块（组长）

**配置字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |

| provider | text | 服务提供商 |

| is_enabled | boolean | 是否启用 |

- 连接测试
